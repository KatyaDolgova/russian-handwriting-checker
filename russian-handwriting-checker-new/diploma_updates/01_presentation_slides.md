СЛАЙД 11 — «Как работает проверка: CheckService»
================================================================================

ЗАГОЛОВОК: Как работает проверка: CheckService
ПОДЗАГОЛОВОК: Routes → CheckService → LLMService → parse JSON → highlight errors

--- СХЕМА (стрелки) ---
POST /check/stream
  → FunctionRepository.get(id)
  → _build_messages() [system + user]
  → LLMService.stream()
  → _parse() [JSON → dict, очистка markdown-обёртки ```json```]
  → _build_result()

--- ТАБЛИЦА «что входит в итоговый ответ» ---
corrected_text   | Исправленный текст
errors           | Список ошибок (JSON-массив)
score            | Числовая оценка
score_max        | Максимальный балл (из функции)
pass_fail        | «зачёт» / «незачёт» (для итогового сочинения)
criteria         | Баллы по критериям ОГЭ / ЕГЭ (JSON-объект)
min_words        | Минимальное кол-во слов (из настроек функции)
html_highlighted | Текст с <span> вокруг ошибок
comment          | Общий комментарий

--- КОД (оставь прежний _highlight) ---
def _parse(self, raw: str) -> dict:
    # Очищаем markdown-обёртку ```json ... ``` если модель её добавила
    cleaned = re.sub(r'^```(?:json)?\s*', '', raw.strip())
    cleaned = re.sub(r'\s*```\s*$', '', cleaned)
    match = re.search(r'\{[\s\S]*\}', cleaned)
    if not match:
        return {}
    try:
        return json.loads(match.group(0))
    except json.JSONDecodeError:
        fixed = re.sub(r',\s*([\]}])', r'\1', match.group(0))
        try:
            return json.loads(fixed)
        except json.JSONDecodeError:
            return {}

--- БЛОК 4: Версионирование ---
